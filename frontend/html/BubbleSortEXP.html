Aim
Objective of this experiment

To understand and visualise the working of the Bubble Sort algorithm — one of the simplest comparison-based sorting techniques — and analyse its step-by-step behaviour, including comparisons, swaps, and pass-by-pass progress.

Learning Objective: Develop an intuitive understanding of how Bubble Sort repeatedly compares adjacent elements and moves larger values toward the end of the array — similar to how bubbles rise to the surface of water.

By the end of this experiment, you will be able to trace the execution of Bubble Sort on any given array, count comparisons and swaps, identify sorted sub-arrays, and understand its O(n²) time complexity.

Description
Theory & background

Bubble Sort is an elementary sorting algorithm that repeatedly steps through the list, compares adjacent elements, and swaps them if they are in the wrong order. This pass is repeated until the list is sorted.

The algorithm gets its name because smaller elements gradually "bubble" to the top of the list (beginning), while larger elements "sink" to the bottom (end) with each pass.

Program in C language:

#include <stdio.h> // Function to perform bubble sort on an array void Bubble_sort(int a[], int n) { int flag = 0; // Variable to check if any swapping happened for (int i = 0; i < n - 1; i++) { flag = 0; // Reset flag for each pass for (int j = 1; j < n - i; j++) { if (a[j - 1] > a[j]) { // Swap the elements flag = 1; int temp = a[j - 1]; a[j - 1] = a[j]; a[j] = temp; } } // If no swapping happened, the array is already sorted if (flag == 0) break; } } int main() { int n; if (scanf("%d", &n) != 1) return 0; int a[n]; for (int i = 0; i < n; i++) { scanf("%d", &a[i]); } // Call the bubble sort function Bubble_sort(a, n); // Print the sorted array for (int i = 0; i < n; i++) { printf("%d ", a[i]); } return 0; }

Procedure
Step-by-step instructions

Follow each step carefully to understand how Bubble Sort operates on the array.

  1. Start with an unsorted array of n elements displayed as bubbles on screen.
  2. Begin Pass 1: Compare the first two adjacent elements (index 0 and 1).
  3. If the left element is greater than the right element, swap them. Otherwise, move on without swapping.
  4. Move to the next pair (index 1 and 2) and repeat the comparison. Continue until the end of the unsorted portion.
  5. After Pass 1, the largest element will have "bubbled up" to the last position. Mark it as sorted.
  6. Begin Pass 2: Repeat the process for the remaining n−1 elements. The second-largest element settles in its position.
  7. Continue for n−1 passes total. After each pass, one more element is placed in its final sorted position.
  8. The algorithm is complete when no swaps occur in a pass, or after n−1 passes — whichever comes first.
  9. Record the total number of comparisons and swaps made during the sort in the Result section.
