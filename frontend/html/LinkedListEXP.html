Singly Linked List

Aim
Objective of this experiment

To understand and visualise the fundamental operations of a Singly Linked List — a dynamic data structure where each node stores a value and a pointer to the next node — and analyse insertions at the head, tail, and arbitrary positions, as well as deletion and search operations step-by-step.

Learning Objective: Develop an intuitive understanding of how nodes are linked through pointers, how the head pointer anchors the list, and how dynamic memory allocation enables flexible list growth — unlike fixed-size arrays.

By the end of this experiment, you will be able to trace insertions and deletions on any linked list, understand pointer re-assignment, and appreciate the O(n) traversal cost versus the O(1) head-insertion advantage of linked lists.

Description
Theory & background

A Singly Linked List is a linear data structure where each element (called a node) contains two parts: a data field that stores the value, and a next pointer that holds the address of the subsequent node. The list is anchored by a head pointer; the last node's next pointer is NULL, signalling the end of the list.

Unlike arrays, linked lists do not require contiguous memory. Nodes are dynamically allocated and connected purely through pointers, making insertions and deletions at the head extremely efficient — O(1) — while random access still requires O(n) traversal.

Program in C language:

#include <stdio.h> #include <stdlib.h> // Node structure struct Node { int data; struct Node* next; }; // Insert at head void insertHead(struct Node** head, int val) { struct Node* newNode = (struct Node*)malloc(sizeof(struct Node)); newNode->data = val; newNode->next = *head; *head = newNode; } // Insert at tail void insertTail(struct Node** head, int val) { struct Node* newNode = (struct Node*)malloc(sizeof(struct Node)); newNode->data = val; newNode->next = NULL; if (*head == NULL) { *head = newNode; return; } struct Node* temp = *head; while (temp->next != NULL) temp = temp->next; temp->next = newNode; } // Delete by value void deleteNode(struct Node** head, int val) { struct Node* temp = *head, *prev = NULL; while (temp != NULL && temp->data != val) { prev = temp; temp = temp->next; } if (temp == NULL) { printf("Value not found\n"); return; } if (prev == NULL) *head = temp->next; else prev->next = temp->next; free(temp); } // Search by value — returns index (0-based) or -1 int search(struct Node* head, int val) { int idx = 0; while (head != NULL) { if (head->data == val) return idx; head = head->next; idx++; } return -1; } // Display list void display(struct Node* head) { while (head != NULL) { printf("%d -> ", head->data); head = head->next; } printf("NULL\n"); } int main() { struct Node* head = NULL; insertTail(&head, 10); insertTail(&head, 20); insertTail(&head, 30); insertHead(&head, 5); display(head); // 5 -> 10 -> 20 -> 30 -> NULL deleteNode(&head, 20); display(head); // 5 -> 10 -> 30 -> NULL printf("Search 10: index %d\n", search(head, 10)); return 0; }

Search: O(n)

Procedure
Step-by-step instructions

Follow each step carefully to understand how a Singly Linked List responds to insertions, deletions, and searches.

  1. Observe the initial state — an empty list shown with only a NULL terminator and no head node.
  2. Use Insert at Head to add a node. Notice the new node becomes the head and its next pointer links to the former head.
  3. Use Insert at Tail to append a node. Observe the traversal to the last node before the new node is attached.
  4. Use Insert at Position by specifying a value and an index. Watch the list traverse to position i−1, then re-wire pointers to splice the new node in.
  5. Use Delete by Value. Observe the traversal that locates the target node, then the pointer bypass that removes it from the chain.
  6. Use Search to find a value. Watch the traversal node-by-node and note the index when the value is found (or "Not Found").
  7. Observe the Event Log throughout — it records every insertion, deletion, traversal step, and pointer update.
  8. After each operation, check the Result tab for an updated operations summary table and the current list state.
Tip: Use the Speed slider to slow down the animation so you can clearly see pointer re-assignment happening node by node during traversal.
