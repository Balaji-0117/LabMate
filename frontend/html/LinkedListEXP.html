Aim
Objective of this experiment

To understand and visualise the fundamental operations of a Singly Linked List — a dynamic data structure where each node stores a value and a pointer to the next node — and analyse insertions at the head, tail, and arbitrary positions, as well as deletion, search, traversal, and reversal operations step-by-step.

Learning Objective: Develop an intuitive understanding of how nodes are linked through pointers, how the head pointer anchors the list, and how dynamic memory allocation enables flexible list growth — unlike fixed-size arrays.

By the end of this experiment, you will be able to trace insertions, deletions, traversals, and pointer reversals on any linked list — and appreciate the O(1) head-insertion advantage versus the O(n) traversal cost.

Description
Theory & background

A Singly Linked List is a linear data structure where each element (called a node) contains two parts: a data field that stores the value, and a next pointer that holds the address of the subsequent node. The list is anchored by a head pointer; the last node's next pointer is NULL, signalling the end of the list.

Unlike arrays, linked lists do not require contiguous memory. Nodes are dynamically allocated and connected purely through pointers, making insertions and deletions at the head extremely efficient — O(1) — while random access still requires O(n) traversal.

Program in C language:

linked_list.c C
#include <stdio.h> #include <stdlib.h> struct Node { int data; struct Node* next; }; void insertHead(struct Node** head, int val) { struct Node* newNode = malloc(sizeof(struct Node)); newNode->data = val; newNode->next = *head; *head = newNode; } void insertTail(struct Node** head, int val) { struct Node* newNode = malloc(sizeof(struct Node)); newNode->data = val; newNode->next = NULL; if (*head == NULL) { *head = newNode; return; } struct Node* temp = *head; while (temp->next != NULL) temp = temp->next; temp->next = newNode; } void deleteNode(struct Node** head, int val) { struct Node* temp = *head, *prev = NULL; while (temp && temp->data != val) { prev = temp; temp = temp->next; } if (!temp) return; if (!prev) *head = temp->next; else prev->next = temp->next; free(temp); } int search(struct Node* head, int val) { int idx = 0; while (head) { if (head->data == val) return idx; head = head->next; idx++; } return -1; } struct Node* reverse(struct Node* head) { struct Node *prev = NULL, *curr = head, *nextTemp; while (curr) { nextTemp = curr->next; curr->next = prev; prev = curr; curr = nextTemp; } return prev; }

Procedure
Step-by-step instructions

Follow each step carefully to understand how a Singly Linked List responds to every operation.

  1. Click Tutorial in the simulation to walk through the linked list railway metaphor — each node is a coach, each pointer is a coupler.
  2. Use Begin+ to insert at the head. Notice the new coach becomes the locomotive and its coupler links to the former first coach.
  3. Use End+ to append a coach. Watch the simulation traverse every coach until it reaches the caboose, then attaches the new one.
  4. Use Pos+ with a value and index to insert mid-list. Observe coupler breaking and reconnection animating in real time.
  5. Use Begin−, End−, or Pos− to delete. Watch the targeted coach shake and vanish, with couplers reconnecting around the gap.
  6. Use 🔍 Search to find a value. Each coach lights up as it is visited; the result coach turns gold when found.
  7. Use ▶ Traverse to walk the entire list. Every node is highlighted in sequence until NULL.
  8. Use ↺ Reverse for the highlight — watch prev (purple), curr (yellow), and nextTemp (cyan) pointer badges animate the true in-place reversal algorithm, then coaches physically reorder.
  9. Monitor the Pseudocode panel — the current executing line is highlighted as you watch operations unfold.
  10. Check the Memory Map panel to see each node's address and next-pointer update in real time.
Tip: Hover over any coach to see a tooltip with its full node details — data, address, and next pointer value.

✓ Conclusion

1
What is a Linked List?

A Linked List is a linear data structure in which elements are stored in nodes, and each node contains data and a pointer to the next node.

2
What are the advantages of Linked List over arrays?

Linked Lists allow dynamic memory allocation and easy insertion and deletion of elements without shifting data.

3
What are the disadvantages of Linked List?

Linked Lists require extra memory for pointers and do not support direct random access like arrays.

4
What is the role of a pointer in a Linked List?

A pointer stores the address of the next node and helps in connecting nodes together in the Linked List.

5
What is the difference between Linked List and Array?

Arrays store elements in contiguous memory locations and support direct access, whereas Linked Lists store elements in separate nodes connected using pointers.
