Aim
Objective of this experiment

To understand and visualise the working of the Merge Sort algorithm — a highly efficient, comparison-based, divide-and-conquer sorting technique — and analyse its step-by-step behaviour, including the divide phase, merge phase, and the reconstruction of the sorted array.

Learning Objective: Develop an intuitive understanding of how Merge Sort recursively divides an array into halves, sorts each half independently, and then merges them back together in sorted order — visualising how the divide-and-conquer strategy guarantees an efficient and stable sort.

By the end of this experiment, you will be able to trace the full recursive execution of Merge Sort on any given array, understand the divide and merge steps, count comparisons, and appreciate its guaranteed O(n log n) time complexity across all cases.

Description
Theory & background

Merge Sort is a divide-and-conquer algorithm that recursively splits an array into two halves, sorts each half, and then merges the two sorted halves into a single sorted array. It was invented by John von Neumann in 1945 and remains one of the most widely used sorting algorithms today.

The algorithm works in two distinct phases: the Divide phase, where the array is split down to individual elements (which are trivially sorted), and the Merge phase, where sorted sub-arrays are combined by repeatedly picking the smaller of the two front elements.

Program in C language:




#include <stdio.h>

// Merge two sorted halves: A[low..mid] and A[mid+1..high]
void Merge(int A[], int n, int low, int mid, int high) {
    int i = low;
    int j = mid + 1;
    int B[high - low + 1];
    int k = 0;

    while (i <= mid && j <= high) {
        if (A[i] < A[j]) {
            B[k++] = A[i++];
        } else {
            B[k++] = A[j++];
        }
    }

    while (i <= mid) {
        B[k++] = A[i++];
    }

    while (j <= high) {
        B[k++] = A[j++];
    }

    k = 0;
    for (int i = low; i <= high; i++) {
        A[i] = B[k++];
    }
}

// Recursive merge sort function
void MergeSort(int A[], int n, int low, int high) {
    if (low >= high) return;

    int mid = (low + high) / 2;

    MergeSort(A, n, low, mid);
    MergeSort(A, n, mid + 1, high);

    Merge(A, n, low, mid, high);
}

int main() {
    int n;

    if (scanf("%d", &n) != 1) return 0;

    int A[n];

    for (int i = 0; i < n; i++) {
        scanf("%d", &A[i]);
    }

    MergeSort(A, n, 0, n - 1);

    for (int i = 0; i < n; i++) {
        printf("%d ", A[i]);
    }

    return 0;
}

Procedure
Step-by-step instructions

Follow each step carefully to understand how Merge Sort divides and merges the array.

  1. Start with an unsorted array of n elements displayed as blocks on screen.
  2. Divide Phase — Step 1: Find the midpoint of the array and split it into two halves: the left sub-array and the right sub-array.
  3. Divide Phase — Step 2: Recursively apply the same split to each half. Continue dividing until every sub-array contains exactly one element (a single element is always sorted).
  4. Once all sub-arrays are of size 1, the Merge Phase begins. Take two adjacent sorted sub-arrays.
  5. Compare the front elements of both sub-arrays. Place the smaller element into the merged output array. Advance the pointer in the sub-array from which the element was taken.
  6. Repeat the comparison until all elements from one sub-array are exhausted. Then copy any remaining elements from the other sub-array directly into the output.
  7. The merged sub-array is now sorted. Repeat the merge process for progressively larger sub-arrays, working your way back up the recursion tree.
  8. After log₂(n) levels of merging, the entire array is merged into a single fully sorted array.
  9. Record the total number of comparisons and merge levels in the Result section.
Tip: Unlike Bubble Sort, Merge Sort never actually swaps elements — it builds a new sorted sequence by selecting and placing elements during the merge step. Pay close attention to the merge phase in the Simulation to see this clearly.
